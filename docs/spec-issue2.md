## 元 Issue

> 記録のタイミングで、現在入力しているものが暫定何位かを表示するようにしたい

## 前提(ヒアリング結果に基づく仮定)

ヒアリングで以下を確認済み:

1. 順位づけの対象範囲は **カテゴリ内の全商品横断の単価比較**
   (`docs/spec.md` の「カテゴリ内比較」画面・`rankByUnitPrice` と同じ考え方)
2. 順位の対象期間・条件(直近 N ヶ月・特売込み)は既存の「底値」設定と揃える。
   なお設定画面にある項目は対象期間(`bottomWindowMonths`)のみで、
   「特売込み」は既存のデフォルト挙動(`excludeSale` 未指定)を踏襲する意味である
3. 表示タイミングは、商品・価格・内容量が揃った時点での**リアルタイム表示**
   (保存前の確認画面ではなく、入力中に随時更新)

以下 4〜8 は当初仮定として起票し、2026-07-14 のレビュー・ヒアリングで確認済み:

4. 順位の算出対象は「カテゴリ内の商品ごとの基準単位あたり単価」。
   ただし**入力中の商品自身**は、既存の底値ではなく**今入力している値(暫定単価)**を
   その商品の代表値として使う。他の商品は既存の底値(設定期間・特売込み)をそのまま使う
   → 「**入力中の価格そのものが、カテゴリ内で何位相当の安さか**」を答える指標にする。
   (注: 実際に保存した場合の商品の底値は `min(既存底値, ドラフト)` になるため、
   ドラフトが自商品の既存底値より高いときは、保存後の実順位はこの表示より良くなる。
   本機能はあくまで「この価格の安さ」を答えるものであり、保存後順位の予測ではない)
5. 記録のない他商品は比較対象から除外する(`rankByUnitPrice` の既存挙動を踏襲)。
   また、記録はあるが底値の単価が算出不能(`bottomPrice` が `unitPrice: null` の
   フォールバック結果を返す商品)も、単価比較ができないため**比較対象・母数から除外**する
   (この点は既存 `rankByUnitPrice` が null を末尾に含めて数える挙動と異なるため、
   流用時に注意する)
6. 単位が baseUnit に換算不能な入力(許容外の単位)の間は順位を計算せず非表示にする。
   なお `RecordPage` の単位セレクタは `allowedUnits(baseUnit)` に制限されているため
   UI 上はほぼ発生しない。関数レベルの防御的仕様として定める
7. カテゴリ内に比較対象(他商品の底値)が 1 件もない場合は「比較対象なし」の表示にする
   (0位/1位のような紛らわしい表示をしない)
8. 同順位(単価が完全一致)の場合は**同順位として上に寄せる**。
   すなわち順位は「ドラフト単価より**厳密に安い**比較対象の数 + 1」で定める
   (例: 2位の商品と同額なら「暫定2位」と表示する)。同率であることの明示表記は行わない
9. この順位表示は既存の「価格を記録」画面(`RecordPage`)にのみ追加する。
   商品詳細・ホーム等の既存画面への変更は伴わない

## Objective

### 何を作るか

価格記録画面(`RecordPage`)で、商品・価格・内容量を入力している最中に、
「今入力している単価だと、このカテゴリ内で暫定何位か」をリアルタイムに表示する。
店頭でその場で「これは記録する価値がある安さか」を判断しやすくする。
順位計算に店舗は不要のため、**店舗が未選択でも表示する**。

### 誰のためか

- 店頭で価格を記録する買い物客(既存ユーザーストーリーの延長)

### ユーザーストーリー

- 買い物客として、価格記録画面で価格・内容量を入力しながら、
  それがカテゴリ内で何位相当の安さかをその場で確認したい
- 買い物客として、比較対象がまだ無い(カテゴリ内に他の記録が無い)場合は、
  順位ではなくその旨がわかる表示を見たい

## ロジック仕様

- 新規関数(仮称)`rankDraftInCategory`:
  - 入力: カテゴリ内の全商品、全価格記録、対象商品 ID、入力中のドラフト値
    (`price` / `quantity` / `unit`)、`BottomPriceOptions`(既存の底値設定を渡す)
  - ドラフト値の有効条件: `price > 0` かつ `quantity > 0` かつ単位が baseUnit に換算可能。
    無効な間は `null` を返す(保存時バリデーションと同じ下限条件を揃える)
  - 処理:
    1. 対象商品以外の各商品について、既存の `bottomPrice`(設定期間・特売込み)を計算。
       記録が無い商品、および底値の単価が算出不能(`unitPrice: null`)の商品は
       比較対象から除外する(前提 5)
    2. 対象商品については、ドラフト値から `calcUnitPrice` で単価を算出
       (無効なら `null` を返して呼び出し側で非表示にする)
    3. 比較対象が 0 件なら `{ kind: 'noCandidates' }` を返す
    4. 順位 = 「ドラフト単価より厳密に安い比較対象の数 + 1」(前提 8、同額は同順位)。
       総数 = 比較対象の数 + 1(**対象商品自身を含む**。よって常に `rank <= total`)
  - 戻り値(判別可能なユニオン):
    - `{ kind: 'ranked'; rank: number; total: number }` … 順位が算出できた
    - `{ kind: 'noCandidates' }` … ドラフトは有効だが比較対象が 0 件
    - `null` … ドラフト値が無効(価格・内容量が未入力/0 以下、単位不整合)で順位計算不可

## UI 仕様(RecordPage)

- 商品・価格・内容量が入力済み(いずれも 0 より大きい)で単位換算が可能なとき
  (= `kind: 'ranked'`)、「このカテゴリで暫定 {rank} 位 / {total} 商品中」のように表示する。
  店舗の選択有無は表示条件に含めない
- ドラフトは有効だが比較対象が無いとき(= `kind: 'noCandidates'`)は、
  「カテゴリ内に比較できる記録がありません」のように**その旨を表示する**
  (0位/1位のような紛らわしい順位表示はしない)
- 入力が不足している間(= `null`)は、順位表示を出さない(既存レイアウトを崩さない)
- 入力値が変わるたびに再計算して即時反映する(保存ボタン押下は不要)

## Testing Strategy

`docs/spec.md` の Testing Strategy を継承。本機能では以下を追加する:

- 単体テスト(`bottomPrice.test.ts`): 
  - ドラフト値が既存の他商品の底値より安い/高い場合の順位
  - ドラフト単価が他商品の底値と完全一致した場合、その商品と同順位になる(前提 8)
  - カテゴリ内に比較対象(他商品の記録)が無い場合に `{ kind: 'noCandidates' }` を返す
  - ドラフトの単位が baseUnit に換算不能な場合に `null` を返す
  - `price` または `quantity` が 0 以下の場合に `null` を返す
  - 底値の単価が算出不能(全記録の単位が不整合)な他商品は、順位・総数の両方から
    除外される(前提 5)
  - 同一商品の既存底値とドラフト値が異なっても、ランキングには常にドラフト値が使われる
  - `total` に対象商品自身が含まれる(比較対象 N 件のとき total = N + 1)
  - `options`(`windowMonths` / `excludeSale`)が他商品の底値計算に反映される
- コンポーネントテスト(`RecordPage`):
  - 入力途中(価格未入力等)は順位が表示されないこと、入力が揃うと表示されること
  - 比較対象が無い場合に「比較対象なし」の表示が出ること
  - 順位計算に `book.bottomWindowMonths ?? DEFAULT_BOTTOM_WINDOW_MONTHS` が
    渡されること(設定画面の底値設定との一致)
