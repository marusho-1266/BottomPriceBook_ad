# Spec: Issue #21 オンボーディング(初回起動時のチュートリアル)

> Status: **Draft(ヒアリング確定・実装前)** / 作成: 2026-07-20
> 対象 Issue: [#21 オンボーディング](https://github.com/marusho-1266/BottomPriceBook_ad/issues/21)
> 親仕様: `docs/spec.md`

## ヒアリング結果(2026-07-20 確定)

1. **表示形式**: スライド式ウォークスルー。初回ログイン直後にモーダルで数枚(4枚)のカードを表示し、「次へ」で進める(画面内コーチマークは採用しない)
2. **説明範囲**: 主要4機能の概要のみを1枚ずつ軽く紹介する(各画面の全操作を詳しく説明する詳細ガイドは作らない)
   - 1枚目: 価格を記録する(店頭でその場で入力)
   - 2枚目: ホームで底値を確認する
   - 3枚目: カテゴリ内比較で商品横断の単価比較をする
   - 4枚目: 底値帳を共有する(招待コード)
3. **再表示**: 設定画面に「使い方を見る」の導線を追加し、いつでも再表示できるようにする

## 前提(仮定 — 誤りがあれば指摘してください)

1. **表示トリガー**: `App.tsx` の `Gate` で `bookReady` になった直後(book 作成・購読開始後、`HomePage` 初回表示のタイミング)にオーバーレイとして表示する。認証・メール確認・book 準備前には表示しない
2. **既読管理**: `localStorage` に `uid` ごとのキーで既読フラグを保存する(`BookProvider.tsx` の `storageKey(uid)` と同様のパターン)。既読なら自動表示しない。サインアウト→別アカウントでログインしても、そのアカウント固有の既読状態になる(Firestore には保存しない。デバイス間の同期は不要な軽量機能のため)
3. **スキップ可能**: 各カードに「スキップ」導線(右上「×」または「スキップ」テキスト)を常設し、いつでも閉じられる。閉じた時点で既読フラグを立てる(最後まで見なくても再表示しない)
4. **設定画面からの再表示**: 「使い方を見る」タップ時は既読フラグの書き換えは行わない(既に既読の状態を維持したまま、同じモーダルを手動で開くだけ)
5. **UI 実装**: 新規コンポーネント `src/features/onboarding/OnboardingModal.tsx` を作成し、`PickerSheet.tsx` 等と同様に画面下からのシート/中央モーダルとしてオーバーレイ表示する。ドット状のページインジケーターと「戻る/次へ」ボタン、最終ページのみ「はじめる」ボタンで閉じる
6. **コンテンツはハードコード**: 4枚のスライド内容(タイトル・説明文・アイコン)はコンポーネント内に定数として定義する。CMS 等の外部管理は不要(将来内容を変える場合もコード変更で対応する規模)
7. **画像・イラストは使わない**: 既存の Lucide アイコン(例: 電卓/レシート系アイコン、ホーム、比較、共有アイコン)+ 見出しテキストで構成する。新規アセット(画像・SVG イラスト)は追加しない(依存追加なし・実装コスト最小化のため)
8. **Analytics**: `docs/spec-issue18.md` の既存方針に倣い、以下のカスタムイベントを追加する(PII なし)
   - `onboarding_shown`(自動表示時)
   - `onboarding_completed`(最終ページまで到達して閉じた場合)
   - `onboarding_skipped`(途中でスキップ・閉じた場合)
   - `onboarding_reopened`(設定画面からの手動再表示時)
9. **既存ユーザーへの影響**: 既にログイン済みの既存ユーザーはこの機能リリース後の初回アクセス時に(既読フラグが存在しないため)オンボーディングが表示される。これは許容する(意図的にリリース後全員に一度見せる)
10. **アクセシビリティ**: モーダル表示中は背景の操作を `ConfirmDialog.tsx` 等と同様にオーバーレイでブロックし、Esc キー/背景タップでは閉じない(誤操作防止。閉じるには明示的なスキップ/はじめるボタンのみ)

## Objective

### 何を作るか

初回ログインしたユーザーに対し、アプリの主要4機能(価格記録・底値確認・カテゴリ内比較・共有)を
スライド形式のモーダルで簡潔に紹介するオンボーディング機能を追加する。設定画面からいつでも
再表示できる。

### 背景・課題

Issue より:「初回起動時のチュートリアル。現状は身内に口頭説明する前提の作り」。
現状は開発者が身内(知り合い)に口頭で使い方を説明する前提でリリースされており、
説明なしで初めて触るユーザーには価格記録・底値の見方・単価比較・共有といった
主要な使い方が伝わらない。アプリ内で完結する最小限の案内を用意することで、
口頭説明なしでも基本的な使い方が伝わるようにする。

### ユーザーストーリー

- 初めてアプリを開いたユーザーとして、口頭説明を受けなくても主要機能の概要を知りたい
- ユーザーとして、オンボーディングが不要だと感じたらすぐにスキップしたい
- 一度スキップ・完了したユーザーとして、あとから設定画面で使い方を見返したい

## Tech Stack

追加の依存パッケージなし。既存の React 19 + `localStorage` + Firebase Analytics(既存 `trackEvent`)のみで実装する。

## Commands

既存: `npm run lint` / `npm run test` / `npm run build`(追加コマンドなし)

## Project Structure(追加分)

```
src/features/onboarding/
  OnboardingModal.tsx   → スライド UI(ページインジケーター・戻る/次へ/スキップ/はじめる)
  content.ts            → 4 枚分のスライド内容(タイトル・説明文・アイコン)定数
  storage.ts            → 既読フラグの localStorage 読み書き(uid ごとのキー)
```

`App.tsx` の `Gate`(`bookReady` 判定後)に組み込み、`SettingsPage.tsx` に「使い方を見る」の
`SettingsLink` 相当の導線を追加する。

## Code Style

親仕様 `docs/spec.md` から変更なし。`BookProvider.tsx` の `storageKey(uid)` パターンを踏襲し、
既読フラグの key 生成を関数化する。

```ts
// features/onboarding/storage.ts
const KEY_PREFIX = 'onboarding-seen:';

export function hasSeenOnboarding(uid: string): boolean {
  return localStorage.getItem(KEY_PREFIX + uid) === '1';
}

export function markOnboardingSeen(uid: string): void {
  localStorage.setItem(KEY_PREFIX + uid, '1');
}
```

## Testing Strategy

| レベル | 対象 |
|---|---|
| 単体 | `storage.ts` の既読フラグ読み書き(uid ごとに独立していること) |
| コンポーネント | `OnboardingModal.tsx`: 「次へ」でページが進むこと、最終ページで「はじめる」を押すと `onComplete` が呼ばれること、スキップで `onSkip` が呼ばれること、ページインジケーターの表示位置が正しいこと |
| 手動 | 初回ログイン時に自動表示されること、設定画面の「使い方を見る」から再表示できること、2 回目以降のログインでは自動表示されないこと |

## Boundaries

- **Always**
  - 新規 npm 依存・新規画像アセットを追加しない(既存の Lucide アイコンとテキストのみで構成)
  - モーダルはユーザーの明示的な操作(スキップ/はじめる)でのみ閉じる
  - Analytics イベントに PII を含めない
- **Ask first**
  - オンボーディングの内容を Firestore 等でリモート管理する方式に変える場合
  - 画面内コーチマーク(要素をハイライトする方式)を追加する場合
  - デバイス間で既読状態を同期する(Firestore 保存)場合
- **Never**
  - 認証・メール確認・book 準備が完了する前にオンボーディングを表示しない(表示条件の前提が崩れるため)

## Success Criteria

- [ ] 初回ログイン(book 準備完了直後)でオンボーディングモーダルが自動表示される
- [ ] 4 枚のスライドを「次へ/戻る」で移動でき、最終ページに「はじめる」ボタンがある
- [ ] 任意のページで「スキップ」して閉じられる
- [ ] 一度閉じる(スキップ or 完了)と、次回以降のログインでは自動表示されない
- [ ] 設定画面の「使い方を見る」から、既読状態に関わらずいつでも再表示できる
- [ ] `onboarding_shown` / `onboarding_completed` / `onboarding_skipped` / `onboarding_reopened` の Analytics イベントが記録される
- [ ] 既存テスト(`npm run test` 等)がすべて通る
