## ヒアリング結果(2026-07-22 確定)

1. **UI 方針**: 案A「底値司令塔ダッシュボード」(サマリー + 底値一覧テーブル + 右ペイン詳細)
2. **端末判定**: 画面幅ベース。**タブレットは PC 寄り**に扱う(スマホ専用レイアウトは狭い幅のみ)
3. **スコープ**: **(α)** PC ホームの案Aダッシュボード + 認証後全画面のサイドナビシェル
4. **右ペイン**: 閲覧のみ。編集・削除は従来どおり商品詳細ページ(`/products/:id`)から行う
5. **サマリー 3 枚**: 現行ホームと同じ定義・算出を維持(登録商品 / 今週の記録 / 底値更新)
6. **詳細導線**: 右ペインに「詳細を開く」を置き、`/products/:id` へ遷移する

## 前提(仮定 — 誤りがあれば指摘してください)

1. **スマホ UI は維持**する。狭い幅では現行の `max-w-md` + 下部タブバーをそのまま使う。店頭利用の体験を壊さない
2. **判定は画面幅のみ**とする。User-Agent 判定は使わない。リサイズでレイアウトが切り替わってよい
3. **ブレークポイント**: Tailwind `md`(min-width: **768px**)以上を PC レイアウト、未満をスマホレイアウトとする(タブレットを PC 寄りにするため `lg` ではなく `md`)
4. **ルートは共通**する。`/pc` 等の別 URL は作らない。同じパスでシェルとホーム見た目だけ切り替える
5. **PC シェルの対象**は `AppShell` 配下の認証後画面(ホーム・記録・比較・設定・カテゴリ/店舗設定・商品詳細)。ログイン・利用規約・招待参加(`/join`)は現行どおり(PC 専用サイドナビは付けない)
6. **記録・比較・設定の中身**は本 Issue では本格再設計しない。サイドナビ付きのワイド表示にし、既存コンポーネントを横幅に収まるよう配置する程度とする(記録の電卓 UI も当面そのまま可)
7. **右ペインの選択状態**はホーム画面内のローカル UI 状態とする。URL には載せない(一覧の選択と「詳細を開く」遷移を分離する)
8. **新規 API / Firestore データモデル変更なし**。既存の products / priceRecords / categories / stores / `bottomPrice` を組み合わせて表示する
9. **デザイン**は既存トークン(クリーム + オレンジ、M PLUS Rounded、Lucide)を踏襲する。PC 用に別テーマは作らない
10. **強制切替トグル**(「常にスマホ表示」等)は本 Issue に含めない
11. **モック HTML**(`docs/pc-ui-samples/`)は仕様検討用の静的資料であり、本番コードには取り込まない

## Objective

### 何を作るか

幅の広い端末(タブレット含む)向けに、サイドナビ付きの PC シェルと、
ホームの「底値司令塔」ダッシュボード(サマリー・一覧テーブル・右ペイン詳細)を追加する。
狭い幅では現行のモバイル UI を維持する。

### 背景・課題

Issue より:「PC用のページを作成する事は可能か / 起動端末から判断は可能なのか」。
現状は `AppShell` が `max-w-md` 固定のため、PC でも中央の狭いカラム表示のみ。
家での俯瞰・比較・確認には横幅を活かせておらず、本格 PC レイアウトが求められている。
店頭はスマホのまま、家ではタブレット/PC でダッシュボード的に使う、という役割分担がゴール。

### ユーザーストーリー

- PC / タブレット利用者として、サイドナビでホーム・記録・比較・設定を行き来したい
- 家で開いたとき、登録商品数・今週の記録・底値更新と、底値一覧を一目で俯瞰したい
- 一覧で商品を選び、右ペインで底値・店舗別・履歴の概要を確認したい
- 編集や詳しい操作が必要なら、「詳細を開く」から従来の商品詳細ページへ進みたい
- 店頭のスマホでは、これまでどおり下部タブのモバイル UI を使いたい

## Tech Stack

追加の依存パッケージなし。既存の React 19 + React Router + Tailwind CSS v4 + Lucide で実装する。
レイアウト切替は CSS ブレークポイント(`md:`)および必要なら `window.matchMedia('(min-width: 768px)')` を用いる。

## 画面仕様

### 共通: レイアウト切替

| 幅 | シェル | ホーム |
|---|---|---|
| `< 768px` | 現行ボトムタブ + `max-w-md` | 現行カード一覧 |
| `≥ 768px` | 左サイドナビ + メイン(幅上限あり可) | 案Aダッシュボード |

サイドナビ項目(現行タブと対応):

| ラベル | パス | 備考 |
|---|---|---|
| ホーム | `/` | |
| 記録 | `/record` | 現行は中央 FAB。PC では通常ナビ項目でよい |
| 比較 | `/compare` | |
| 設定 | `/settings` | |

アクティブ状態は `NavLink` の isActive で示す。ブランド(そこねこ)表示をサイド上部に置く。

### PC ホーム(案A)

1. **ヘッダー領域**
   - タイトル「ホーム」(または現行相当の book 切替 `BookSwitcher` をメイン上部に配置)
   - 商品名検索(現行と同じフィルタ: 商品名の部分一致)
   - 「＋ 価格を記録」→ `/record` への導線(任意だが推奨)

2. **サマリー 3 枚**(現行ロジックをそのまま流用)
   - **登録商品**: `products.length`
   - **今週の記録**: 直近 7 日以内の `priceRecords` 件数
   - **底値更新**: 今週の記録のうち、現在の商品底値レコードと id が一致する件数
     (`HomePage.tsx` の `weekRecords` / `bottomUpdatedCount` と同一)

3. **左: カテゴリ別・底値一覧テーブル**
   - 行: 商品名、底値価格、特売バッジ、店舗名、基準単位あたり単価
   - カテゴリ見出しでグルーピング(現行ホームと同様。カテゴリ内比較へのリンクがあってもよい)
   - 行クリック(または行選択)で右ペインに当該商品を表示
   - 底値がない商品は現行どおり一覧に出さない

4. **右: 選択商品の概要ペイン**(閲覧のみ)
   - 底値ヒーロー(価格・店舗・特売バッジ・商品名・内容量/単価)
   - 店舗別底値(概要)
   - 最近の履歴(件数は実装時に妥当な上限、例: 直近 5〜10 件)
   - **「詳細を開く」ボタン** → `/products/:id` へ遷移
   - **編集・削除 UI は置かない**(商品詳細ページの責務)
   - 未選択時はプレースホルダ(「一覧から商品を選択」等)

5. **空状態**
   - 記録が 0 件のときは現行と同様の案内文 + 記録への導線

### 他画面(記録・比較・設定・商品詳細)

- PC シェル(サイドナビ)の内側に既存ページを表示する
- 本 Issue の必須は「横幅を活かして窮屈でないこと」。電卓や比較ロジックの再設計は将来スコープ
- 商品詳細は従来どおり編集・削除の正式な場所とする

## Code Style

親仕様 `docs/spec.md` から変更なし。例:

```tsx
/** Tailwind md(768px) 以上を PC レイアウトとみなす */
export function useIsDesktopLayout(): boolean {
  // matchMedia('(min-width: 768px)') を購読し、リサイズに追従する
}
```

- レイアウト分岐はシェルとホームに閉じ、各 feature の API / 底値ロジックは触らない
- サマリー算出は `HomePage` から純粋関数へ切り出してもよい(テスト容易性のため)。挙動は現行と同一であること

## Testing Strategy

| レベル | 対象 |
|---|---|
| 単体 | サマリー算出を切り出した場合、現行と同じ件数になること |
| コンポーネント | PC ホーム: 行選択で右ペインが更新されること、「詳細を開く」で `/products/:id` へ進むこと、右ペインに編集・削除がないこと |
| コンポーネント | シェル: md 相当でサイドナビ、未満でボトムタブが出ること(jsdom + matchMedia モック、またはクラス分岐のスナップショット相当) |
| 手動 | 768px 前後でリサイズしレイアウトが切り替わること、タブレット幅でサイドナビ + 案Aホームになること、スマホ幅で現行 UI のままであること、記録・比較・設定がサイドナビから開けること |

## Boundaries

- **Always**
  - 狭い幅のモバイル UI(ボトムタブ・現行ホーム)を壊さない
  - サマリー 3 指標の定義を現行から変えない
  - 右ペインは閲覧と「詳細を開く」のみ。編集・削除は商品詳細ページへ誘導
  - 新規 npm 依存を追加しない
  - Firestore / Functions / セキュリティルールを変更しない
- **Ask first**
  - ブレークポイントを 768px 以外に変える場合
  - 記録・比較画面の本格的な PC 再設計を本 Issue に含める場合
  - `/pc` など別ルート方式に変える場合
  - 「常にモバイル表示」等の強制トグルを追加する場合
- **Never**
  - User-Agent だけでレイアウトを固定しない
  - 検討用 HTML モックを本番バンドルにコピーして使わない
  - スマホ幅でサイドナビのみ・ボトムタブなし、にしない
